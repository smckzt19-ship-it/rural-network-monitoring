# PRD / Prompt: Одностраничный дашборд мониторинга строительства оптических сетей

## Контекст
Необходимо создать одностраничный сайт для мониторинга строительства и подключения сетей в сельских населенных пунктах Республики Казахстан. Сайт должен показывать общую картину по проекту, прогресс строительства, технологии подключения, этапы СМР и статус выполнения графика по регионам.

Данные на первом этапе могут быть демонстрационными, но структура должна позволять заменить их на реальные данные без переделки интерфейса.

## Цель продукта
Дать руководителю проекта и региональным координаторам быстрый визуальный обзор:

- сколько всего сельских населенных пунктов вошло в проект;
- в скольких СНП уже построены оптические сети;
- какие технологии используются;
- какие этапы СМР выполнены и сколько времени занимает каждый этап;
- какие регионы опережают график, идут по графику или отстают.

## Основные пользователи
- Руководитель проекта.
- Региональные координаторы.
- Технический надзор.
- Заказчик или представитель госоргана, которому нужна сводная картина.

## Основные показатели
На первом экране должны быть видны ключевые метрики:

- всего СНП в проекте: около 3000;
- количество СНП, где сети построены;
- процент построенных сетей от общего объема;
- количество регионов, которые опережают график;
- количество регионов, которые отстают;
- количество регионов, которые идут по графику.

## Разрез по регионам
Нужно показать региональную картину по Республике Казахстан.

Для каждого региона отображать:

- название региона;
- количество СНП в проекте;
- количество СНП с построенными сетями;
- процент выполнения;
- плановый процент выполнения;
- отклонение от плана в процентных пунктах;
- статус: `Опережаем`, `По графику`, `Отстаем`.

## Визуализация регионов
Сделать схематичную карту или региональную диаграмму.

Требования к графику:

- все регионы отображаются круглыми маркерами;
- все круги должны быть одного размера;
- цвет круга показывает статус:
  - зеленый: опережаем;
  - синий: по графику;
  - красный: отстаем;
- внутри круга показывать короткое название региона и процент выполнения;
- полное название, количество СНП, построено, план и статус показывать в подсказке при наведении;
- график должен быть читаемым на десктопе и мобильных экранах;
- подписи не должны выходить за пределы кругов.

## Технологии подключения
Нужно показать, по какой технологии подключены населенные пункты.

Технологии:

- ВОЛС;
- Wifi-public;
- Спутник.

Визуализация:

- круговая или кольцевая диаграмма;
- рядом список технологий с количеством СНП и процентом;
- сумма по технологиям должна совпадать с количеством СНП, где сети построены.

## Этапы СМР
Строительство делится на этапы:

1. Подготовительные работы.
2. Прокладка оптического кабеля.
3. Монтажные и сварочные работы.
4. Измерения и паспортизация.
5. Сдача объекта в эксплуатацию.

Для каждого этапа нужно показывать:

- процент выполнения;
- плановое время выполнения;
- фактическое время выполнения;
- статус по срокам: быстрее плана, в норме, дольше плана.

## Детализация этапов
Этапы СМР включают следующие работы:

### 1. Подготовительные работы
- Входной контроль кабеля и комплектующих.
- Проверка целостности.
- Проверка соответствия спецификациям.
- Замер затухания сигнала.
- Подготовка трассы.
- Очистка кабельных каналов.
- Расчистка просек при воздушной прокладке.
- Подготовка траншей при подземной прокладке.

### 2. Прокладка оптического кабеля
- В грунте: рытье траншей, укладка в ЗПТ, засыпка с сигнальной лентой.
- В кабельной канализации: затяжка кабеля в существующие трубы.
- По опорам: подвеска самонесущего оптического кабеля или навивка на грозотрос.
- Внутри зданий: прокладка в лотках, коробах или монтаж по стенам.

### 3. Монтажные и сварочные работы
- Разделка кабеля.
- Снятие защитных оболочек, брони и модульных трубок.
- Зачистка волокон.
- Скалывание прецизионным скалывателем.
- Сварка оптоволокна автоматическим сварочным аппаратом.
- Монтаж муфт и оптических кроссов.
- Укладка сваренных волокон в защитные гильзы.

### 4. Измерения и паспортизация
- Тестирование линии оптическим рефлектометром OTDR.
- Измерения на разных длинах волн.
- Выявление потерь.
- Проверка качества сварных соединений.
- Оформление исполнительной документации.
- Подготовка рефлектограмм и протоколов измерений.

### 5. Сдача объекта в эксплуатацию
- Формирование приемочной комиссии.
- Проверка соответствия построенной линии проекту и нормам.
- Подписание актов выполненных работ.
- Передача объекта на баланс.

## Фильтры и взаимодействие
На странице должны быть:

- фильтр по статусу: все, опережаем, по графику, отстаем;
- поиск по региону;
- интерактивные подсказки на региональном графике;
- таблица регионов, которая синхронизируется с фильтрами;
- визуальное приглушение регионов, которые не подходят под выбранный фильтр.

## Визуальный стиль
Интерфейс должен выглядеть как рабочий аналитический дашборд, а не как лендинг.

Требования:

- спокойный деловой стиль;
- хорошая читаемость;
- карточки с ключевыми метриками;
- аккуратная сетка;
- цвета статусов должны быть понятными и стабильными;
- без лишней декоративности;
- адаптивность под мобильные и десктопные экраны.

## Технические требования
На первом этапе сайт может быть статическим:

- `index.html`;
- `styles.css`;
- `script.js`;
- без обязательного backend;
- данные можно хранить в JavaScript-массивах;
- структура данных должна быть простой для замены на реальные данные из API или CSV.

## Критерии приемки
Результат считается готовым, если:

- на странице видна общая картина по 3000 СНП;
- есть разбивка по регионам;
- есть визуальное различие статусов `Опережаем`, `По графику`, `Отстаем`;
- региональный график использует круглые маркеры одинакового размера;
- есть разбивка по технологиям строительства;
- есть этапы СМР с плановым и фактическим временем;
- таблица регионов показывает прогресс и отклонение от плана;
- фильтры и поиск работают;
- страница корректно открывается как одностраничный сайт.
